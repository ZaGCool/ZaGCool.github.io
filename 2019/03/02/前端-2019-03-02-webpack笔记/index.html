    webpack 4.0+简介

    webpack中文网

    webpack官网

    webpack是一个前端自动化打包构建工具

    基于nodeJs平台的工具/模块 会遵循nodeJs的规范

    nodeJs ===> npm 包管理器

    为什么要使用webpack

    • 代码开发过程中出现的大量注释空格以及换行导致文件体积过大，webpack可以通过压缩打包的方式来解决这些问题
    • ES6在开发过程中非常便捷方便我们开发，但是运行在不同的浏览器中，可能会出现兼容性问题，webpack可以帮助我们在打包的时候，把高版本语法转换为低版本语法，实现兼容
    • 在组件化开发过程中，资源整合会导致index.html入口引入文件过多，不方便管理，webpack可以帮助通过nodeJs或者es6的模块化规范来解决资源整合问题，还可以处理vue的单文件组件
    • webpack还有其他的诸如启动本地服务，实现模块热加载，区分开发环境和生产环境，对各个类型文件进行预编译…

    webpack可以使用nodeJs的一个规范

    commonJS规范 （主要解决了模块的导入导出方式）

    还支持es6的模块化规范

    Javascript

    webpack的loader以及plugins

    webpack 可以使用 loader 来预处理文件。这允许你打包除 JavaScript 之外的任何静态资源。你可以使用 Node.js 来很简单地编写自己的 loader。

    webpack 有着丰富的插件接口(rich plugin interface)。webpack 自身的多数功能都使用这个插件接口。这个插件接口使 webpack 变得极其灵活。

    ES6模块化

    在之前的 javascript 中一直是没有模块系统的，前辈们为了解决这些问题，提出了各种规范, 最主要的有CommonJS和AMD两种。前者用于服务器，后者用于浏览器。而 ES6 中提供了简单的模块系统，完全可以取代现有的CommonJS和AMD规范，成为浏览器和服务器通用的模块解决方案。
